# MVP Scope Guard

## Goal
Keep the project focused and prevent feature creep.

## Checklist
- Does this feature support the core user flow?
- Is it necessary for a demoable MVP?
- Does it add backend, UX, or testing complexity out of proportion?
- Can it be postponed without hurting the core value?

## Output format
- Decision: Keep in MVP / Postpone
- Reason: 2–4 lines
- Impact on scope: low / medium / high
